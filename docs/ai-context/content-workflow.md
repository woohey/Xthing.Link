# Xthing.Link 内容工作流

## 目标

让写作过程尽量简单，同时保持：
- 一份主稿
- 网站为完整版本
- 外部平台便于分发

## 推荐流程

1. 在 Obsidian 中编写原始 Markdown 主稿
2. 使用统一 frontmatter 整理进入站点内容目录
3. 在网站上发布完整版本
4. 针对头条、知乎、小红书等平台生成派生版本
5. 在文章 metadata 中记录已同步的平台链接

## 原则

- Obsidian 是唯一主稿源
- 网站是主站和完整归档
- 外部平台不做机械复制，而做适配改写
- 不在第一阶段追求全自动一键同步

## 建议维护的 metadata

- `canonical`
- `platforms`
- `source`
- `status`
- `summary`
- `cover`

这些字段主要服务于：
- 内容管理
- 多平台分发记录
- 站内展示
- 后续自动化扩展
