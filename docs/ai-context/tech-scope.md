# Xthing.Link — 技术范围与部署形态

本文档确认站点技术选型与阿里云交付方式，作为工程与运维的单一事实来源。

## 前端框架（已确认）

- **Astro（静态优先）**：内容页（Blog、Projects、About 等）以静态 HTML 为主，利于 SEO 与首屏性能。
- **React Islands（按需交互）**：仪表盘、可视化等需要客户端交互的页面或区块，以 Astro 中的 React 岛（islands）形式挂载，仅在必要处加载 JS。

该组合满足「以静态内容为主、部分 Demo 需要复杂交互」的长期维护目标，且与下文部署形态一致。

## 阿里云部署（已确认）

- **形态**：**纯静态托管**。在构建环境执行 Astro 生产构建，将输出目录 **`dist/`** 作为站点根目录发布。
- **Web 服务器**：在自管 ECS 或同类环境上，使用 **Nginx** 或 **Caddy** 将域名指向 `dist/`（或同步/upload 后的等价静态目录），无需为默认站点运行 Node 常驻进程。
- **子路径**：Demo 与同站内容一并构建，例如 ` /demos/<project>` 与主站同一 `dist/`，由同一虚拟主机配置提供。

## 非目标（第一阶段）

- 默认不依赖 Next.js 全栈或独立 Node 服务承载主站。
- 动态数据优先：静态 JSON / 预计算；确需实时 API 时再单独扩展，不在本技术范围内默认假设。
