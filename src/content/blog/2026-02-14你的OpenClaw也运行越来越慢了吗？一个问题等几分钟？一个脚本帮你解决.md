# 你的 OpenClaw 也越来越慢了吗？

> 一个问题等几分钟，不是模型慢，是你配置有问题

---

今天跟小渡对话，一个问题要等好几秒甚至几分钟。第一反应是网络问题，后来一看 status，傻眼了：

**102k/262k tokens (39%)**

 main session 已经吃了 10 万 token，不卡才怪。

---

 🔍 问题在哪

运行 `openclaw status`，发现两个坑：

**1. Token 爆炸**
- 当前 session 占用 102k tokens
- 历史消息累积太多

**2. Sessions 堆积**
- 10 个 active sessions
- 很多是几天前的僵尸 session（2d, 4d, 5d, 7d...）

简单说：**过去聊天的内容全带着，能不慢吗？**

---

 ⚡ 立即优化（6步走）

# 1. 清理旧 Sessions
```bash
openclaw sessions cleanup --older-than 2d
```
马上释放资源。

# 2. 重置当前 Session
在对话里发：
```
/reset
```
清空历史，token 归零（文件记忆还在）。

# 3. 关闭无用 Channel
Discord 配置了但没 token，占着茅坑不拉屎：
```bash
openclaw config set channels.discord.enabled false
```

# 4. 拉宽 Heartbeat
默认 30 分钟一次检查，太频繁：
```bash
openclaw config set heartbeat.interval "2h"
```

# 5. 限制历史消息
编辑 `~/.openclaw/agents/main/agent.yaml`：
```yaml
context:
  maxHistory: 20      # 只留最近 20 条
  maxTokens: 50000    # 封顶 5 万 token
```

# 6. 自动清理
编辑 `~/.openclaw/config.yaml`：
```yaml
sessions:
  maxAge: "24h"       # 24 小时无活动自动清
  maxCount: 5         # 最多留 5 个 session
```


脚本会帮你：
1. 清理 2 天前的旧 sessions
2. 关闭 Discord
3. 设置 heartbeat 为 2 小时
4. 添加自动清理配置
5. 重启 gateway

**执行完，响应速度提升 50% 以上。**

---

 💡 核心认知

OpenClaw 不是慢，是**你让它记了太多东西**。

就像跟人聊天，每次开口前要先背一遍之前聊过的所有内容，能不慢吗？

**原则：定期清理，轻装上阵。**
